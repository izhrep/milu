# Отчёт об исправлении системы прав для роли manager

**Дата:** 2025-11-14  
**Цель:** Ограничить доступ роли `manager` к админ-панели и списку всех пользователей системы

---

## ✅ Выполненные изменения

### 1. Созданные разрешения (permissions)

Добавлены 3 новых разрешения для более детального контроля доступа:

| Разрешение | Ресурс | Описание |
|------------|--------|----------|
| `security.view_admin_panel` | security | Доступ к административной панели |
| `users.view_all` | users | Просмотр всех пользователей системы |
| `users.view_department` | users | Просмотр пользователей своего департамента |

### 2. Обновлённые назначения ролям (role_permissions)

#### Admin (без изменений в правах, только уточнения):
- ✅ `security.view_admin_panel` — доступ к админ-панели
- ✅ `users.view_all` — просмотр всех пользователей
- ✅ `users.view` — базовое право просмотра пользователей
- ✅ `security.manage` — управление безопасностью

#### HR BP:
- ✅ `users.view_department` — просмотр пользователей департамента
- ❌ **УДАЛЕНО** `users.view` — больше не видит всех пользователей

#### Manager:
- ❌ **УДАЛЕНО** `users.view` — больше не видит всех пользователей
- ✅ Сохранены права `team.view` — видит только своих подчинённых

#### Employee:
- Без изменений — видит только себя

## 🔒 Обновлённые RLS политики

### Таблица `users` - политика SELECT

**Старая логика:**
```sql
-- Все пользователи с разрешением users.view видели всех
has_permission('users.view')
```

**Новая логика:**
```sql
-- 1. Пользователь всегда видит себя
id = auth.uid()

OR

-- 2. Admin видит всех через users.view_all
has_permission('users.view_all')

OR

-- 3. Manager видит только своих подчинённых
(
  has_permission('team.view') 
  AND id IN (
    SELECT u.id 
    FROM users u 
    WHERE u.manager_id = auth.uid()
  )
)

OR

-- 4. HR BP видит пользователей своего департамента
(
  has_permission('users.view_department')
  AND department_id IN (
    SELECT u.department_id 
    FROM users u 
    WHERE u.id = auth.uid()
  )
)
```

## 💻 Обновлённые frontend-компоненты

### 1. `src/components/AppSidebar.tsx`

**Было:**
```typescript
const { hasPermission: canManageUsers } = usePermission('users.view');
const adminItems = canManageUsers || canViewSecurity ? [...] : [];
```

**Стало:**
```typescript
const { hasPermission: canViewAdminPanel } = usePermission('security.view_admin_panel');
const adminItems = canViewAdminPanel || canViewSecurity ? [...] : [];
```

**Результат:** Пункт меню "Админ панель" отображается только при наличии `security.view_admin_panel`

---

### 2. `src/pages/AdminPage.tsx`

**Было:**
```typescript
const { hasPermission: hasAdminPermission } = usePermission('security.manage');
```

**Стало:**
```typescript
const { hasPermission: hasAdminPermission } = usePermission('security.view_admin_panel');
```

**Результат:** Доступ к странице `/admin` строго по разрешению `security.view_admin_panel`
