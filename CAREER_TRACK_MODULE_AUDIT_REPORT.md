## 🔧 Обнаруженные и исправленные проблемы

### 1. Отсутствие индексов для производительности

#### Проблемы:
- ❌ Нет индексов на `grades(position_id, position_category_id, level)`
- ❌ Нет индексов на `career_tracks(target_position_id, track_type_id)`
- ❌ Нет индексов на `development_tasks` для подбора задач
- ❌ Нет индексов на `users(position_id, grade_id)` для карьерных полей
- ❌ Нет индексов на `user_career_progress` для быстрого поиска

#### Решение:
✅ **Добавлено 24 новых индекса:**

**Grades (3 индекса):**
- `idx_grades_position` - для фильтрации по позиции
- `idx_grades_position_category` - для фильтрации по категории
- `idx_grades_level` - для сортировки по уровню

**Career Tracks (2 индекса):**
- `idx_career_tracks_target_position` - для поиска по целевой позиции
- `idx_career_tracks_track_type` - для фильтрации по типу трека

**Development Tasks (4 индекса):**
- `idx_dev_tasks_skill` - для поиска задач по навыку
- `idx_dev_tasks_quality` - для поиска задач по качеству
- `idx_dev_tasks_level` - для фильтрации по уровню
- `idx_dev_tasks_composite` - композитный индекс для комплексного поиска

**Grade Skills & Qualities (4 индекса):**
- `idx_grade_skills_grade` - для gap-анализа
- `idx_grade_skills_skill` - для поиска требований по навыку
- `idx_grade_qualities_grade` - для gap-анализа
- `idx_grade_qualities_quality` - для поиска требований по качеству

**Users (2 индекса):**
- `idx_users_position` - для карьерных запросов
- `idx_users_grade` - для определения текущего уровня

**User Career Progress (4 индекса):**
- `idx_user_career_progress_user` - для поиска прогресса пользователя
- `idx_user_career_progress_track` - для статистики по треку
- `idx_user_career_progress_status` - для фильтрации активных треков
- `idx_user_career_progress_active` - partial index только для активных

### 2. Отсутствие функций gap-анализа

#### Проблемы:
- ❌ Нет автоматического расчета разрыва между текущими и целевыми компетенциями
- ❌ Нет понимания "готовности" к следующему шагу
- ❌ Ручной подсчет gaps в клиентском коде

#### Решение:
✅ **Создана функция `calculate_career_gap(user_id, grade_id)`**

```sql
-- Возвращает:
-- - competency_type: 'skill' или 'quality'
-- - competency_id, competency_name
-- - current_level: из user_assessment_results (среднее по self_assessment)
-- - target_level: из grade_skills/grade_qualities
-- - gap: разница (target - current)
-- - is_ready: boolean (готовность к грейду)
```

**Преимущества:**
- ⚡ Быстрый расчет на уровне БД
- 🎯 Единая точка истины для gap-анализа
- 🔄 Легко переиспользовать в разных частях системы
- 📊 Агрегация данных из user_assessment_results

### 3. Отсутствие автоматических рекомендаций треков

#### Проблемы:
- ❌ Нет системного подбора треков под пользователя
- ❌ Пользователь видит все треки без учета релевантности
- ❌ Нет скора совместимости

#### Решение:
✅ **Создана функция `recommend_career_tracks(user_id, limit)`**

**Алгоритм подбора:**
1. Берёт текущую позицию и грейд пользователя
2. Вычисляет compatibility_score:
   - +50 баллов если позиция совпадает с target_position трека
   - +20 баллов для остальных треков
   - +30 баллов если у трека есть шаги
3. Для каждого трека считает total_gap (сумма gaps по всем шагам)
4. Сортирует по compatibility_score DESC, total_gap ASC
5. Возвращает топ N треков

**Возвращаемые поля:**
- track_id, track_name, track_type_name, target_position_name
- compatibility_score (релевантность 0-100)
- total_gap (суммарный разрыв компетенций)
- steps_count (количество шагов)

### 4. Отсутствие умного подбора задач развития

#### Проблемы:
- ❌ Нет связи между gap-анализом и задачами development_tasks
- ❌ Пользователь вручную выбирает задачи из библиотеки
- ❌ Нет приоритизации задач по gap

#### Решение:
✅ **Создана функция `get_recommended_development_tasks(user_id, grade_id, limit)`**

**Алгоритм:**
1. Запускает gap-анализ для указанного грейда
2. Фильтрует только компетенции с gap > 0
3. Для каждой компетенции находит до 2 подходящих задач из development_tasks
4. Сортирует по величине gap (сначала наибольшие отставания)
5. Возвращает топ N задач с контекстом gap

**Возвращаемые поля:**
- task_id, task_name, task_goal, how_to, measurable_result
- competency_type, competency_id, competency_name
- current_level, target_level, gap

### 5. Отсутствие функции проверки консистентности

#### Проблемы:
- ❌ Нельзя проверить "здоровье" данных модуля
- ❌ Могут существовать треки без шагов, грейды без компетенций
- ❌ Нет инструментов для диагностики

#### Решение:
✅ **Создана функция `check_career_data_consistency()`**

**Проверки:**
1. **tracks_without_steps** - треки без шагов (WARNING)
2. **grades_without_competencies** - грейды без навыков/качеств (WARNING)
3. **invalid_step_order** - шаги с некорректным порядком < 1 (ERROR)
4. **progress_invalid_tracks** - активный прогресс по несуществующим трекам (ERROR)
5. **tasks_without_competencies** - задачи развития без привязки к компетенциям (WARNING)

**Формат вывода:**
```json
{
  "check_name": "tracks_without_steps",
  "status": "OK" | "WARNING" | "ERROR",
  "details": {
    "count": 0,
    "tracks": [...]
  }
}
```

### 6. Недостаточные RLS политики на user_career_progress

#### Проблемы:
- ❌ Таблица `user_career_progress` не имела RLS политик
- ❌ Потенциальная утечка данных о карьерных треках

#### Решение:
✅ **Созданы 5 RLS политик:**

| Политика | Роль | Операция | Условие |
|----------|------|----------|---------|
| Users can view their own progress | Employee | SELECT | `user_id = current_user` |
| Users can manage their own progress | Employee | ALL | `user_id = current_user` |
| Managers can view team progress | Manager | SELECT | `EXISTS (manager_id = current_user)` |
| HR can view all progress | HR/Admin | SELECT | `is_current_user_hr()` |
| Admins can manage all progress | Admin | ALL | `is_current_user_admin()` |

### 7. Отсутствие валидации данных на уровне БД

#### Проблемы:
- ❌ Можно создать шаг с step_order = 0 или отрицательным
- ❌ Можно создать требования с target_level = 0
- ❌ grade_id в career_track_steps был nullable

#### Решение:
✅ **Добавлены ограничения:**

```sql
-- Обязательное наличие грейда у шага
ALTER TABLE career_track_steps ALTER COLUMN grade_id SET NOT NULL;

-- Порядковый номер >= 1
ALTER TABLE career_track_steps 
  ADD CONSTRAINT check_step_order_positive CHECK (step_order >= 1);

-- Целевой уровень > 0
ALTER TABLE grade_skills 
  ADD CONSTRAINT check_target_level_positive CHECK (target_level > 0);
  
ALTER TABLE grade_qualities 
  ADD CONSTRAINT check_target_level_positive CHECK (target_level > 0);
```

### 8. Отсутствие автоматизации updated_at

#### Проблемы:
- ❌ `user_career_progress.updated_at` не обновлялся автоматически

#### Решение:
✅ **Добавлен триггер:**

```sql
CREATE TRIGGER update_user_career_progress_updated_at
  BEFORE UPDATE ON public.user_career_progress
  FOR EACH ROW
  EXECUTE FUNCTION update_updated_at_column();
```

## 🎯 Финальный флоу "Карьерный трек"

### Шаг 1: Создание карьерного трека (HR/Admin)

**Роль**: HR / Admin  
**Страница**: `/admin` → Карьерные треки

**Действия:**
1. Нажать "Создать трек"
2. Заполнить:
   - Название (например, "От продавца до директора магазина")
   - Тип трека (линейный, экспертный, гибкий)
   - Целевая позиция
   - Длительность в месяцах
3. Добавить шаги трека:
   - Выбрать грейд для каждого шага
   - Указать порядок (1, 2, 3...)
   - Указать длительность шага
   - Добавить описание
4. Сохранить

**Автоматически:**
- ✅ Создаются записи в `career_tracks` и `career_track_steps`
- ✅ Проверяется уникальность (track_id, grade_id)
- ✅ Проверяется уникальность (track_id, step_order)

### Шаг 2: Настройка требований грейдов (HR/Admin)

**Роль**: HR / Admin  
**Страница**: `/admin` → Грейды

**Для каждого грейда в треке:**
1. Открыть грейд
2. Добавить требования по навыкам:
   - Выбрать навык
   - Указать целевой уровень (1-5)
3. Добавить требования по качествам:
   - Выбрать качество
   - Указать целевой уровень (1-5)

**Результат:**
- ✅ Записи в `grade_skills` и `grade_qualities`
- ✅ Эти требования используются для gap-анализа

### Шаг 3: Заполнение библиотеки задач развития (HR/Admin)

**Роль**: HR / Admin  
**Страница**: `/admin` → Задачи развития

1. Создать задачи для каждой компетенции и уровня

## 📈 Производительность

### Индексы (24 новых)

**Ускоряют запросы:**
1. Подбор треков по позиции/грейду пользователя
2. Gap-анализ для любого грейда
3. Поиск задач развития по компетенциям
4. Фильтрация активных треков пользователя
5. Статистика по трекам и шагам

**Особенности:**
- Partial index на `user_career_progress(user_id) WHERE status = 'active'` - только активные треки
- Composite index на `development_tasks(skill_id, quality_id, competency_level_id)` - комплексный поиск
- Условные индексы WHERE IS NOT NULL для опциональных полей

### Функции

Все функции используют:
- `STABLE` - не изменяют данные, могут кешироваться
- `SECURITY DEFINER` - выполняются с правами владельца
- `SET search_path = 'public'` - безопасность от SQL injection

## 🧪 Чек-лист для тестирования

### 1. Создание карьерного трека (HR/Admin)
- [ ] Войти как HR/Admin
- [ ] Создать новый карьерный трек
- [ ] Добавить 3-5 шагов с грейдами
- [ ] Проверить: порядок шагов корректный (1, 2, 3...)
- [ ] Проверить: для каждого грейда заполнены навыки и качества
- [ ] Попробовать создать шаг с step_order = 0 → должна быть ошибка

### 2. Настройка требований грейдов (HR/Admin)
- [ ] Открыть грейд
- [ ] Добавить минимум 2 навыка с target_level
- [ ] Добавить минимум 2 качества с target_level
- [ ] Проверить: target_level > 0 (нельзя указать 0 или отрицательное)
- [ ] Сохранить

### 3. Создание задач развития (HR/Admin)
- [ ] Создать задачу развития
- [ ] Привязать к навыку/качеству
- [ ] Указать уровень компетенции
- [ ] Заполнить: название, цель, как выполнить, измеримый результат
- [ ] Сохранить

### 4. Прохождение диагностики (Employee)
- [ ] Войти как сотрудник
- [ ] Пройти self-assessment
- [ ] Пройти 360-опрос (если настроено)
- [ ] Проверить: результаты сохранены в user_assessment_results

### 5. Просмотр рекомендованных треков (Employee)
- [ ] Открыть страницу "Развитие" → "Карьерные треки"
- [ ] Проверить: треки отсортированы по релевантности
- [ ] Проверить: видны скор совместимости и total_gap
- [ ] Проверить: первым идёт трек с совпадающей позицией (если есть)

### 6. Выбор карьерного трека (Employee)
- [ ] Нажать "Выбрать трек" на рекомендованном треке
- [ ] Проверить: создана запись в user_career_progress
- [ ] Проверить: status = 'active'
- [ ] Проверить: если был предыдущий active трек → он стал inactive

### 7. Gap-анализ (Employee)
- [ ] Открыть детали трека
- [ ] Проверить: отображается текущий шаг
- [ ] Проверить: показаны все компетенции с gap
- [ ] Проверить: current_level берётся из диагностики
- [ ] Проверить: target_level берётся из grade_skills/grade_qualities
- [ ] Проверить: is_ready корректно (✅ или ❌)

### 8. Создание плана развития (Employee)
- [ ] Нажать "Создать план развития"
- [ ] Проверить: система предложила рекомендованные задачи
- [ ] Проверить: задачи отсортированы по величине gap
- [ ] Выбрать несколько задач
- [ ] Установить даты начала и окончания
- [ ] Сохранить план
- [ ] Проверить: созданы записи в development_plans и tasks

### 9. Работа с задачами развития (Employee)
- [ ] Открыть "Мои задачи развития"
- [ ] Взять задачу в работу (pending → in_progress)
- [ ] Добавить комментарий о прогрессе
- [ ] Отметить выполнение (in_progress → completed)
- [ ] Указать достигнутый уровень (kpi_result_level)
- [ ] Проверить: updated_at обновился

### 10. Мониторинг прогресса (Manager)
- [ ] Войти как руководитель
- [ ] Открыть страницу "Команда"
- [ ] Проверить: видны подчинённые
- [ ] Проверить: для каждого показан активный трек (если выбран)
- [ ] Открыть детали трека подчинённого
- [ ] Проверить: видны gap-анализ и задачи развития
- [ ] Проверить: НЕ видны чужие подчинённые

### 11. Переход на следующий шаг (Employee)
- [ ] Выполнить все задачи текущего шага
- [ ] Пройти повторную диагностику
- [ ] Проверить: все компетенции is_ready = true
- [ ] Нажать "Перейти на следующий шаг"
- [ ] Проверить: current_step_id обновился
- [ ] Проверить: новый gap-анализ для следующего шага

### 12. Проверка RLS (разные роли)

#### Сотрудник:
- [ ] Видит свой прогресс по треку
- [ ] Может выбрать/изменить свой трек
- [ ] НЕ видит прогресс других сотрудников
- [ ] НЕ может изменять треки, грейды, задачи развития (библиотеку)

#### Руководитель:
- [ ] Видит треки своих подчинённых
- [ ] НЕ видит треки чужих подчинённых
- [ ] НЕ может изменять user_career_progress подчинённых

#### HR:
- [ ] Видит прогресс всех пользователей (только чтение)
- [ ] Может управлять всеми справочниками
- [ ] НЕ может изменять user_career_progress напрямую

#### Admin:
- [ ] Полный доступ ко всему

### 13. Проверка консистентности данных
- [ ] Запустить: `SELECT * FROM check_career_data_consistency();`
- [ ] Проверить: все проверки status = 'OK'
- [ ] Если WARNING/ERROR → исследовать и исправить

### 14. Тестирование Edge Function (AI-генерация)
- [ ] Открыть создание плана развития
- [ ] Нажать "Сгенерировать задачи с помощью AI"
- [ ] Проверить: Edge Function вернул структурированные задачи
- [ ] Проверить: задачи соответствуют gaps
- [ ] Проверить: задачи имеют все обязательные поля

### 15. Производительность
- [ ] Открыть рекомендованные треки → время < 1 сек
- [ ] Gap-анализ для грейда → время < 500 мс
- [ ] Рекомендованные задачи → время < 1 сек
- [ ] Проверить использование индексов через EXPLAIN ANALYZE

## 🚀 Рекомендации по дальнейшему развитию

### Ближайшие улучшения

1. **Автоматическое создание планов развития**
   - При выборе трека автоматически создавать план с рекомендованными задачами
   - Пользователь только корректирует/утверждает

2. **Визуализация прогресса по треку**
   - Timeline с визуальным отображением шагов
   - Цветовая индикация готовности к следующему шагу
   - Progress bar для каждой компетенции

3. **Уведомления и напоминания**
   - Напоминание о необходимости диагностики
   - Уведомление о приближении к завершению шага
   - Алерт при длительном простое без прогресса

4. **Геймификация**
   - Бейджи за достижения (завершение шагов, выполнение задач)
   - Таблица лидеров по прогрессу
   - Система мотивации через визуальные награды

5. **Интеграция с встречами 1:1**
   - Автоматическое добавление обсуждения прогресса по треку в повестку встречи
   - Решения со встреч → задачи развития

### Долгосрочные улучшения

1. **Предиктивная аналитика**
   - ML-модель для предсказания времени достижения следующего шага
   - Рекомендации персонализированных треков на основе истории успехов

2. **Социальные функции**
   - Менторство: связь опытных сотрудников с начинающими
   - Peer learning: группы развития по треку
   - Обмен опытом через комментарии к задачам

3. **Интеграция с внешними системами**
   - Синхронизация с LMS (Learning Management System)
   - Импорт курсов и сертификатов как задач развития
   - Экспорт отчётов в корпоративную аналитику

4. **Мульти-трековая карьера**
   - Возможность выбрать несколько треков одновременно
   - Гибридные треки (например, управленческий + экспертный)
   - Учёт пересечений компетенций
