## 1. Назначение форм (HR-админ)

### 1.1 Управление этапом диагностики
- [ ] HR-админ может запустить этап «Диагностика»
  - [ ] Выбор периода (название этапа)
  - [ ] Выбор списка сотрудников
  - [ ] Указание дат: начало, окончание, дедлайн
  - [ ] Создание этапа с автоматическим назначением участников
- [ ] HR-админ может просматривать список всех этапов
- [ ] HR-админ может редактировать этапы
- [ ] HR-админ может удалять этапы (только без участников)
- [ ] HR-админ может добавлять участников в существующий этап

### 1.2 Управление навыками
- [ ] HR-админ может добавлять новый навык
  - [ ] Название навыка
  - [ ] Описание навыка
  - [ ] Категория навыка
- [ ] HR-админ может редактировать навыки
- [ ] HR-админ может удалять навыки
- [ ] HR-админ может управлять категориями навыков
  - [ ] Создание категории
  - [ ] Редактирование категории
  - [ ] Удаление категории

### 1.3 Управление качествами
- [ ] HR-админ может добавлять новое качество
  - [ ] Название качества
  - [ ] Описание качества
- [ ] HR-админ может редактировать качества
- [ ] HR-админ может удалять качества

### 1.4 Управление вопросами к навыкам
- [ ] HR-админ может добавлять вопросы к навыкам
  - [ ] Текст вопроса
  - [ ] Привязка к навыку
  - [ ] Порядок отображения
- [ ] HR-админ может редактировать вопросы
- [ ] HR-админ может удалять вопросы
- [ ] Вопросы отображаются в форме оценки навыков

### 1.5 Управление вариантами ответов для оценки навыков
- [ ] HR-админ может добавлять варианты ответов (шкала 0-4)
  - [ ] Значение шага (0, 1, 2, 3, 4)

  - [ ] Описание уровня
- [ ] HR-админ может редактировать варианты ответов
- [ ] HR-админ может удалять варианты ответов
- [ ] Варианты ответов отображаются при прохождении опроса

### 1.6 Управление вопросами к качествам (360)

  - [ ] Текст вопроса
  - [ ] Привязка к качеству
  - [ ] Категория вопроса
  - [ ] Поведенческие индикаторы
  - [ ] Порядок отображения
- [ ] HR-админ может редактировать вопросы
- [ ] HR-админ может удалять вопросы
- [ ] Вопросы отображаются в форме оценки 360

### 1.7 Управление вариантами ответов для оценки 360
- [ ] HR-админ может добавлять варианты ответов (шкала 0-4)
  - [ ] Значение (0, 1, 2, 3, 4)

  - [ ] Описание уровня

## 2. Назначение форм (Сотрудник и Руководитель)

### 2.1 Выбор оценивающих (Сотрудник)
- [ ] Сотрудник может выбирать оценивающих для оценки 360
  - [ ] Обязательно: руководитель
  - [ ] Минимум 1 коллега
  - [ ] Добавление коллег из справочника
- [ ] Сотрудник может удалять выбранных оценивающих
- [ ] Сотрудник может заменять оценивающих
- [ ] Есть поиск по справочнику сотрудников
- [ ] Руководитель автоматически добавляется в список оценивающих

### 2.2 Статусы и информация
- [ ] Сотрудник видит статус оценки каждого респондента
  - [ ] "Отправлено на оценку" / "отправлен запрос"
  - [ ] "Оценено" / "выполнено"
- [ ] Сотрудник видит дату окончания этапа Диагностики
- [ ] Сотрудник видит шаги процесса:
  - [ ] Выбор оценивающих
  - [ ] Прохождение оценки
  - [ ] Завершено
- [ ] Отображается информация о периоде диагностики

### 2.3 Функционал для руководителя
- [ ] Руководитель может открывать назначенную форму диагностики сотрудника
- [ ] Руководитель может переключаться между подчинёнными
- [ ] Руководитель может начать оценку из задач/уведомлений
- [ ] Руководитель может оценить сотрудника сразу после начала этапа

## 3. Прохождение формы Оценка 360

### 3.1 Самооценка (Сотрудник)
- [ ] Сотрудник может пройти самооценку 360
  - [ ] Только после начала активного этапа
  - [ ] После выбора оценивающих
- [ ] Сотрудник оценивает каждое качество по шкале 0-4
- [ ] Форма содержит все качества из системы

### 3.2 Оценка качеств сотрудника (Оценивающий/Руководитель)
- [ ] Оценивающий может оценить качества сотрудника по шкале 0-4
- [ ] Руководитель может оценить качества сотрудника по шкале 0-4

### 3.3 Интерфейс формы оценки 360
- [ ] Отображается определение каждого качества
- [ ] Отображаются подсказки к шкале 0-4
- [ ] Есть возможность оставлять комментарий под каждым качеством
- [ ] Есть быстрое перемещение между качествами
- [ ] В шапке формы отображается:
  - [ ] ФИО сотрудника
  - [ ] Должность
  - [ ] Грейд/шаг
  - [ ] Период этапа Диагностики
- [ ] До отправки можно возвращаться к ранее введённым оценкам
- [ ] После отправки форма доступна только для чтения

### 3.4 Комментарии
- [ ] Оценивающий может оставлять комментарии
- [ ] Оценивающий может сделать комментарий анонимным
- [ ] Анонимные комментарии не показывают имя автора
- [ ] Комментарии сохраняются вместе с оценкой

### 3.5 Статус завершения
- [ ] Оценивающий видит подтверждение завершения оценки
- [ ] Оценивающий видит список всех форм на оценку
- [ ] Оценивающий видит статусы по всем формам

## 5. Завершение оценки качеств и формирование Отчёта 360

### 5.1 Доступность отчёта
- [ ] Отчёт 360 доступен после сбора всех оценок:
  - [ ] Самооценка сотрудника
  - [ ] Оценка руководителя
  - [ ] Минимум 1 оценка от коллеги
- [ ] Руководитель может просмотреть отчёт 360 сотрудника

### 5.2 Содержание отчёта 360
- [ ] Отображается усреднённый балл по каждому качеству
- [ ] Напротив каждого балла есть текстовое значение
- [ ] Есть возможность переключаться между качествами
- [ ] Отображается радиальная диаграмма оценки качеств
- [ ] Отображается список групп респондентов и легенда
- [ ] Можно сравнивать оценки по группам:
  - [ ] Я (самооценка)
  - [ ] Руководитель
  - [ ] Коллеги

### 5.3 Детальный просмотр качества
- [ ] Можно раскрыть выбранное качество
- [ ] Отображается описание качества
- [ ] Отображаются поведенческие индикаторы
- [ ] Отображаются оценки по группам оценивших

### 5.4 Комментарии в отчёте
- [ ] Отображаются текстовые комментарии
- [ ] Указано, кто оставил комментарий (кроме анонимных)
- [ ] Анонимные комментарии отображаются без указания автора

## 8. Техническая реализация

### 8.1 База данных
- [ ] Таблица `diagnostic_stages` корректно создаёт этапы
- [ ] Таблица `diagnostic_stage_participants` корректно добавляет участников
- [ ] Уникальные ограничения работают для `(user_id, stage_id)`
- [ ] Уникальные ограничения работают для `(evaluated_user_id, evaluating_user_id)`
- [ ] Триггеры корректно создают задания при добавлении участников
- [ ] RLS-политики разрешают корректный доступ

### 8.2 Триггеры и функции
- [ ] `assign_surveys_to_diagnostic_participant()` создаёт:
  - [ ] Задание на самооценку навыков
  - [ ] Задание на самооценку 360
  - [ ] Задание на оценку от руководителя (если есть)
- [ ] `create_diagnostic_task_for_participant()` создаёт задачу
- [ ] `update_diagnostic_stage_status()` обновляет статус этапа
- [ ] `calculate_diagnostic_stage_progress()` рассчитывает прогресс
- [ ] Триггеры не обращаются к несуществующим полям
- [ ] Все функции используют `get_current_session_user()` вместо `auth.uid()`

### 8.3 Логика назначений
- [ ] При создании участника создаётся самооценка (evaluated = evaluating)
- [ ] При выборе оценивающих создаются отдельные записи в `survey_360_assignments`
- [ ] У одного оцениваемого может быть несколько оценивающих
- [ ] Поле `manager_id` используется только для связей
- [ ] `evaluation_period` корректно устанавливается

### 8.4 Статусы и прогресс
- [ ] Статус assignment корректно обновляется после завершения опроса
- [ ] Статус задачи обновляется после завершения опроса
- [ ] Прогресс этапа корректно рассчитывается
- [ ] Статус этапа меняется в зависимости от прогресса:
  - [ ] `setup` (0%)
  - [ ] `assessment` (0% < x < 100%)
  - [ ] `completed` (100%)
