## Тестовый сценарий (E2E)

### Шаг 1: Самооценка (SELF)
1. Войти под аккаунтом сотрудника
2. Перейти на страницу прохождения оценки `/unified-assessment/:assignmentId`
3. Пройти оценку по навыкам и качествам
4. **Проверить:** для каждого вопроса есть TextArea для комментария
5. **Проверить:** чекбокс "Оставить анонимно" СКРЫТ
6. Оставить комментарии к нескольким вопросам
7. Завершить оценку
8. **Проверить в БД:** `is_anonymous_comment = false` для всех комментариев SELF

### Шаг 2: Оценка руководителем (MANAGER)
1. Войти под аккаунтом руководителя
2. Перейти на страницу оценки подчинённого
3. Пройти оценку
4. **Проверить:** чекбокс "Оставить анонимно" СКРЫТ
5. Оставить комментарии
6. Завершить оценку
7. **Проверить в БД:** `is_anonymous_comment = false` для всех комментариев MANAGER

### Шаг 3: Оценка коллегой (PEER)
1. Войти под аккаунтом коллеги
2. Перейти на страницу оценки 360
3. Пройти оценку
4. **Проверить:** чекбокс "Оставить анонимно" ОТОБРАЖАЕТСЯ
5. **Проверить:** чекбокс по умолчанию ВКЛЮЧЁН
6. Для части вопросов:
   - Оставить комментарий с галочкой "Оставить анонимно" (анонимный комментарий)
   - Оставить комментарий БЕЗ галочки "Оставить анонимно" (неанонимный комментарий)
7. Завершить оценку
8. **Проверить в БД:** 
   - `is_anonymous_comment = true` для анонимных комментариев
   - `is_anonymous_comment = false` для неанонимных комментариев

### Шаг 4: Проверка страницы результатов
1. Войти под аккаунтом сотрудника (или его руководителя, или HR BP)
2. Перейти на `/assessment/results/:userId`
3. **Проверить навыки (Hard Skills):**
   - Под каждой горизонтальной диаграммой навыка отображаются ТОЛЬКО комментарии к этому навыку

   - Под каждой горизонтальной диаграммой качества отображаются ТОЛЬКО комментарии к этому качеству
   - Правила отображения имён аналогичны навыкам
5. **Проверить фильтры:**
   - При изменении фильтров (роль, тип компетенции) комментарии обновляются вместе с диаграммами

## SQL-запросы для проверки

### Проверка структуры таблиц
```sql
-- Проверка наличия полей в hard_skill_results
SELECT column_name, data_type, is_nullable, column_default
FROM information_schema.columns
WHERE table_name = 'hard_skill_results' 
AND column_name IN ('comment', 'is_anonymous_comment');

-- Проверка наличия полей в soft_skill_results
SELECT column_name, data_type, is_nullable, column_default
FROM information_schema.columns
WHERE table_name = 'soft_skill_results' 
AND column_name IN ('comment', 'is_anonymous_comment');
```

### Проверка данных комментариев
```sql
-- Проверка комментариев в hard_skill_results
SELECT 
  hsr.evaluated_user_id,
  hsr.evaluating_user_id,
  hsr.comment,
  hsr.is_anonymous_comment,
  hsr.created_at
FROM hard_skill_results hsr
WHERE hsr.comment IS NOT NULL
ORDER BY hsr.created_at DESC
LIMIT 20;

-- Проверка комментариев в soft_skill_results
SELECT 
  ssr.evaluated_user_id,
  ssr.evaluating_user_id,
  ssr.comment,
  ssr.is_anonymous_comment,
  ssr.created_at
FROM soft_skill_results ssr
WHERE ssr.comment IS NOT NULL
ORDER BY ssr.created_at DESC
LIMIT 20;
```

### Проверка анонимности по типу оценщика
```sql
-- Анализ анонимности комментариев (hard skills)
WITH evaluator_types AS (
  SELECT 
    hsr.id,
    hsr.comment,
    hsr.is_anonymous_comment,
    CASE 
      WHEN hsr.evaluating_user_id = hsr.evaluated_user_id THEN 'SELF'
      WHEN hsr.evaluating_user_id = u.manager_id THEN 'MANAGER'
      ELSE 'PEER'
    END as evaluator_type
  FROM hard_skill_results hsr
  JOIN users u ON u.id = hsr.evaluated_user_id
  WHERE hsr.comment IS NOT NULL
)
SELECT 
  evaluator_type,
  COUNT(*) as total_comments,
  SUM(CASE WHEN is_anonymous_comment THEN 1 ELSE 0 END) as anonymous_count,
  SUM(CASE WHEN NOT is_anonymous_comment THEN 1 ELSE 0 END) as named_count
FROM evaluator_types
GROUP BY evaluator_type;
```
